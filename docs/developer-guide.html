Altuwijri Ticket System

Developer Reference — v2.1 · Last updated 2026-06-28 مرجع المطور — الإصدار 2.1 · آخر تحديث 2026-06-28

Overviewنظرة عامة

The Altuwijri Ticket System (ATS) is an internal IT help-desk platform for the Altuwijri organisation. Employees submit IT support requests; technicians claim, work, and resolve them; managers run reports. It is a LAN-hosted full-stack web application accessible inside the corporate network. نظام تذاكر العتيجي (ATS) هو منصة دعم تقنية معلومات داخلية لمنظمة العتيجي. يقدم الموظفون طلبات دعم تقنية المعلومات؛ يطالب بها الفنيون ويعملون على حلها؛ يستخرج المسؤولون تقارير منها. وهو تطبيق ويب متكامل مستضاف على الشبكة المحلية ويمكن الوصول إليه داخل شبكة الشركة.

Clean Architectureالبنية النظيفة

The backend follows Clean Architecture (also known as Onion / Hexagonal). Dependencies flow strictly inward: outer layers depend on inner ones, never the reverse. تتبع الخلفية نمط البنية النظيفة (المعروفة أيضاً بـ Onion / Hexagonal). تتدفق التبعيات إلى الداخل فقط: الطبقات الخارجية تعتمد على الداخلية وليس العكس.

API Layer TaskFlow.Api
backend/src/TaskFlow.Api/
ASP.NET Core Web API. Controllers, middleware, DI registration, JWT config, SignalR hubs. Contains no business logic.
ASP.NET Core Web API. وحدات التحكم والوسيطات وتسجيل DI وإعداد JWT ومراكز SignalR. لا تحتوي على منطق أعمال.
Application Layer TaskFlow.Application
backend/src/TaskFlow.Application/
All business logic. Services, DTOs, interfaces (IApplicationDbContext, ICurrentUser, IEmailService). Defines what the system does.

backend/src/TaskFlow.Infrastructure/
Implements interfaces. EF Core DbContext, migrations, database seeder, email (SMTP), Identity setup.
تنفيذ الواجهات. EF Core DbContext والترحيلات وبذر قاعدة البيانات والبريد الإلكتروني (SMTP) وإعداد Identity.

Technician Ticket Visibility Ruleقاعدة رؤية التذاكر للفنيين

A technician sees on their board:يرى الفني في لوحته:

  • All tickets assigned to them (any project).جميع التذاكر المعيَّنة له (أي مشروع).
  • All tickets unassigned whose CategoryId matches one of their assigned categories — regardless of project.جميع التذاكر غير المعيَّنة التي يتطابق CategoryId فيها مع إحدى فئاته المعيَّنة — بغض النظر عن المشروع.
  • If the technician has no category assignments, they see all unassigned tickets.إذا لم يكن للفني فئات معيَّنة، يرى جميع التذاكر غير المعيَّنة.

Deployment Guideدليل النشر

Overviewنظرة عامة Two IIS applications on the same server: the Angular SPA at the root, the .NET API at /api. ESET Endpoint Security manages firewall rules — port 80 must be allowed for LAN subnet. تطبيقان IIS على نفس الخادم: Angular SPA على الجذر، .NET API على /api. يدير ESET Endpoint Security قواعد جدار الحماية — يجب السماح بالمنفذ 80 لشبكة LAN الفرعية.

Backend (IIS) Deployنشر الخلفية (IIS)

  1. Touch app_offline.htm in the API folder to gracefully stop IIS from serving requests while files are locked. أنشئ app_offline.htm في مجلد API لإيقاف IIS عن خدمة الطلبات بينما الملفات مقفلة.
    New-Item -ItemType File "D:\Projetcs\Cloude\deploy\api\app_offline.htm" -Force
  2. Publish the .NET project: انشر مشروع .NET:
    dotnet publish src/TaskFlow.Api/TaskFlow.Api.csproj -c Release -o D:\Projetcs\Cloude\deploy\api_pub
  3. Robocopy published output to IIS folder: انسخ المخرجات المنشورة إلى مجلد IIS عبر Robocopy:
    robocopy D:\Projetcs\Cloude\deploy\api_pub D:\Projetcs\Cloude\deploy\api /MIR /NFL /NDL
    Robocopy exit codes 1–3 are success. Only ≥ 4 is an error. أكواد خروج Robocopy 1–3 نجاح. فقط ≥ 4 يعني خطأ.
  4. Remove the offline marker to bring the API back online: احذف ملف الوضع غير المتصل لإعادة API للعمل:
    Remove-Item D:\Projetcs\Cloude\deploy\api\app_offline.htm -Force

Frontend (IIS) Deployنشر الواجهة (IIS)

  1. Add Node to path (if needed in PowerShell session): أضف Node إلى المسار (إذا لزم في جلسة PowerShell):
    $env:Path += ";C:\Program Files\nodejs"
  2. Build Angular: ابنِ Angular:
    cd frontend; node ".\node_modules\@angular\cli\bin\ng.js" build --configuration production
  3. Robocopy the browser output: انسخ مخرجات المتصفح عبر Robocopy:
    robocopy frontend\dist\taskflow\browser D:\Projetcs\Cloude\deploy\web /MIR /NFL /NDL
ESET Firewallجدار الحماية If inbound connections fail on port 80 after deploy, open ESET Endpoint Security → Network protection → Rules, and verify the rule allowing inbound TCP 80 for w3wp.exe is active. إذا فشلت الاتصالات الواردة على المنفذ 80 بعد النشر، افتح ESET Endpoint Security ← حماية الشبكة ← القواعد، وتحقق من تفعيل القاعدة التي تسمح بـ TCP 80 الوارد لـ w3wp.exe.

Database Migrationsترحيلات قاعدة البيانات

  1. Ensure connection string in appsettings.json is correct.تأكد من صحة سلسلة الاتصال في appsettings.json.
  2. Add migration:أضف ترحيلاً: dotnet ef migrations add <Name> -p TaskFlow.Infrastructure -s TaskFlow.Api
  3. Apply:طبّق: dotnet ef database update -p TaskFlow.Infrastructure -s TaskFlow.Api
  4. The seeder (DatabaseSeeder.cs) runs automatically on first start to seed roles and admin user.يعمل البذار (DatabaseSeeder.cs) تلقائياً عند التشغيل الأول لبذر الأدوار والمستخدم المسؤول.
